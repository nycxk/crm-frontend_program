## 1 系统概述

房产CRM系统是一套面向房产租赁业务的全流程管理平台，覆盖**客户获取 → 来访跟进 → 房源匹配 → 签约成交 → 租后退租**的完整业务闭环。

**核心功能模块：**

| 模块 | 说明 |
|------|------|
| 数据概览 | 系统首页，展示核心业务数据 |
| 客户管理 | 客户信息维护、来访记录跟踪 |
| 公海客户 | 超保护期的共享客户资源池 |
| 房产管理 | 房源信息维护、指导价/评估价管理 |
| 渠道管理 | 获客渠道类型及中介公司管理 |
| 成交管理 | 签约成交、退租登记 |
| 联系人 | 独立联系人库，可关联客户 |
| 系统管理 | 人员、部门、角色权限、系统参数配置 |

**业务流程总览：**

```
新增客户 → 记录来访（关联房源+渠道） → 签约成交 → 登记退租
    ↓                                        ↓
  公海认领                              房源状态自动更新
```

## 2 登录与账号安全

### 2.1 登录系统

1. 打开系统网址，进入登录页面
2. 输入**手机号**和**密码**，点击「登录」
3. 登录成功后自动跳转至系统首页

> 支持手机验证码登录（需联系管理员配置）。

### 2.2 修改密码

**方式一：登录后修改**

- 点击系统**右上角用户名**处，找到修改密码入口
- 输入原密码和新密码即可修改

**方式二：忘记密码**

- 在登录页点击「忘记密码」，通过短信验证码重置

### 2.3 初始密码强制修改

当管理员新建人员或重置密码后，该用户密码将恢复为初始密码。**首次登录时系统会自动弹出修改密码窗口**，用户必须修改密码后才能继续使用系统。

> 修改密码后，所有已登录设备均会被强制下线，需重新登录。

### 2.4 退出登录

点击右上角「退出」按钮即可安全退出。退出仅影响当前设备，其他设备仍保持登录状态（除非修改密码）。

### 2.5 会话管理

- 系统默认**允许多端同时在线**（手机、电脑可同时登录）
- 修改密码后，该账号**所有设备**均需重新登录
- 管理员禁用账号后，该账号所有设备将强制下线

## 5 客户管理

### 5.1 客户列表

客户是系统核心数据，每位客户可关联多个联系人、多条来访记录。

#### 查看客户列表

进入「客户管理 → 客户列表」，可看到本人权限范围内的客户列表。

- 支持按**客户姓名**模糊搜索
- 支持按**客户状态**筛选（我的客户 / 租户）
- 列表展示：客户姓名、证件信息、状态、来访次数、关联联系人

#### 新增客户

点击「新增客户」按钮：

1. 填写**客户姓名**（必填）
2. 选填证件类型和证件号（如同时填写，组合须唯一）
3. 可关联已有联系人，或同步新建联系人

> 新增客户默认状态为「我的客户」，记录认领时间用于保护期计算。

#### 客户详情

点击列表中的客户，可查看详细信息，包括：
- 客户基本信息
- 关联联系人列表
- 历史来访记录
- 联系人电话号码（仅本人创建的客户显示完整号码）

#### 编辑与删除

- 可修改客户基本信息和证件信息
- 支持绑定/解绑联系人
- 支持软删除（客户信息保留在数据库中，仅标记删除）

#### 数据权限说明

| 角色/部门 | 可见客户范围 |
|-----------|-------------|
| 系统管理员 | 全部客户 |
| 经营部人员 | 本经营部及下属项目部所有成员的客户 |
| 项目部人员 | 仅本项目部成员的客户 |
| 未分配部门 | 不可见任何客户 |

> 联系人的手机号仅在**本人创建**的客户/联系人记录中显示完整号码，他人创建的记录会脱敏显示（如 `138****8000`）。

---

### 5.2 客户来访

记录每次客户来访/带看信息，是跟进的核心数据。

#### 查看来访记录

进入「客户管理 → 客户来访」，可按以下条件筛选：
- 客户
- 带看房源
- 渠道类型
- 来访日期范围

#### 新增来访记录

点击「新增来访」，填写以下信息：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 来访日期 | 跟进/带看日期 | 是 |
| 关联客户 | 选择已存在的客户 | 是 |
| 带看房源 | 可选择多个房源 | 否 |
| 渠道类型 | 如中介公司、老带新、市场自来、全员营销、泛销 | 否 |
| 渠道实例 | 渠道下的具体来源（如某家中介公司） | 视渠道类型 |
| 详情说明 | 跟进记录文字描述 | 否 |
| 需求清单 | 客户需求（面积、预算等自定义键值对） | 否 |

**渠道实例填写规则：**

| 渠道类型 | 实例填法 |
|----------|----------|
| 中介公司 | 必选一家已录入的中介公司 |
| 老带新 | 可选一个客户作为推荐人 |
| 全员营销 | 可选一个用户作为推荐人 |
| 泛销 | 可选一个联系人 |

> 系统支持来访草稿功能，可暂存未完成的来访记录，后续完善后提交。

---

### 5.3 公海客户

公海是系统中**无归属**客户的共享资源池。客户的「我的客户」状态在超过保护期（默认 30 天）后，会由系统定时任务自动转入公海。

#### 查看公海

进入「公海客户」菜单，可看到所有公海客户。公海中联系人的手机号为**全量脱敏**（全部隐藏）。

#### 认领公海客户

1. 在公海列表中找到想要跟进的客户
2. 点击「认领」按钮
3. 认领成功后，该客户变为你的「我的客户」，可查看完整信息

> 认领后保护期重新计算，你有 30 天时间进行有效跟进。

## 6 房产管理

### 6.1 房产列表

管理所有可租赁的房源信息，每个房源关联一个项目部。

#### 查看房源

进入「房产管理 → 房产列表」，可查看权限范围内的房源。

- 支持按**房源名称**模糊搜索
- 支持按**项目部**筛选
- 支持按**房源状态**筛选（闲置 / 在租）
- 每条房源显示当前最新的指导价和评估价

#### 新增房源

点击「新增房源」，填写：

| 字段 | 说明 |
|------|------|
| 房源名称 | 必填，如「A栋101」 |
| 适租面积 / 非适租面积 / 总面积 | 面积信息 |
| 有证面积 / 无证面积 | 证件面积 |
| 坐落位置 | 地址描述 |
| 所属项目部 | 必选项目部 |
| 所属经营部 | 必选经营部（须为项目部的上级） |
| 指导价 | 必填，价格 + 生效日期 |
| 评估价 | 必填，价格 + 生效日期 |
| 图片 | 可选，上传房源图片 |

> 新建房源默认状态为「闲置」。指导价和评估价支持版本管理，每次调价自动生成新版本。

#### 价格管理

- **单条调价**：编辑房源时修改指导价/评估价，每次修改生成新版本（V1 → V2 → V3...）
- **回退价格**：可将指导价或评估价回退到上一版本
- **批量调价**：见 6.2 节

#### 房源状态说明

| 状态 | 说明 | 触发场景 |
|------|------|----------|
| 闲置 | 可出租 | 新建默认、退租后恢复 |
| 在租 | 已出租 | 签约成交后自动变更 |

---

### 6.2 批量调价

一次性对多个房源调整指导价或评估价。

#### 操作流程

1. 进入「房产管理 → 批量调价」
2. 点击「下载模板」，选择调价类型（指导价 / 评估价）
3. 在 Excel 模板中填写：房源ID、房源名称（核对用）、价格、生效日期、失效日期
4. 上传填好的 Excel 文件，输入批次名称和备注
5. 系统校验通过后一键生效

> 单次最多 5000 行，支持查看历史调价记录和批次回退。

---

### 6.3 内部招租

记录房源对内招租的需求信息，如定向寻找特定业态的租户。

#### 新增招租

填写房源、发起日期、租赁面积、参考价格等信息，可关联看房联系人。

#### 操作说明

- 支持修改招租信息（如停止招租日期）
- 支持软删除
- 招租数据按房源所属项目部做权限隔离

## 7 渠道管理

管理客户来源渠道，渠道分为**渠道类型**和**中介公司（二级）**。

#### 渠道类型

系统预置五种渠道类型：

| 类型 | 是否有二级数据 | 说明 |
|------|---------------|------|
| 中介公司 | 有 | 可挂多家具体中介公司 |
| 老带新 | 无 | 老客户推荐新客户 |
| 市场自来 | 无 | 自然到访 |
| 全员营销 | 无 | 内部员工推荐 |
| 泛销 | 无 | 其他泛化渠道 |

> 支持新增、修改、删除渠道类型。渠道被来访/成交引用后不可删除。

#### 中介公司管理

仅「中介公司」类型渠道下可维护中介公司列表。支持对中介公司进行增删改查，被来访记录引用后不可删除。

## 8 成交管理

管理租赁签约成交的全流程。

#### 新建成交

进入「成交管理」，点击「新建成交」：

| 步骤 | 说明 |
|------|------|
| 选择房源 | 至少选 1 个，必须为闲置状态 |
| 选择客户 | 至少选 1 个，证件号不能为空 |
| 选择渠道 | 必选成交途径（渠道类型 + 实例） |
| 填写金额 | 合同总额 |
| 填写日期 | 签订日期、起租日期、退租日期 |
| 租赁面积 | 可选，默认取所选房源适租面积之和 |

**新建成交后的自动变更：**
- 关联房源状态 → 「在租」
- 关联客户状态 → 「租户」

#### 成交列表

支持按客户、房源、渠道、成交业务员、签订日期范围等多条件筛选。

#### 修改成交

可修改成交信息（已退租的成交不可修改）。修改签订日期或房源时会重新匹配时点指导价/评估价。

#### 登记退租

成交到期或提前退租时，点击「登记退租」，填写实际退租日期。退租后：
- 关联房源恢复为「闲置」
- 若客户无其他在租成交，状态恢复为「我的客户」

> 系统支持成交草稿功能，可暂存不完整的成交信息。

## 9 联系人管理

联系人独立维护，可关联到客户。

#### 新增联系人

填写联系人姓名、手机号（必填，全局唯一）、从属类型、备注。

#### 联系人列表

支持按姓名、手机号搜索。手机号脱敏规则与客户数据权限一致（仅本人创建的联系人显示完整号码）。

#### 绑定到客户

在客户详情页中，可将已有联系人绑定到客户；也支持新建客户时同步新建联系人。

## 10 系统管理

> 系统管理模块通常仅对**系统管理员**和拥有相应权限的角色可见。

### 10.1 人员管理

管理系统的用户账号。

#### 查看人员列表

支持按用户名/手机号/邮箱模糊搜索，按状态筛选。

#### 新增人员

填写用户名、手机号（必填），选择部门、角色，设置状态。**初始密码固定为 `admin123`**，用户首次登录时需强制修改。

#### 编辑人员

可修改用户名、邮箱、部门、角色、状态等。

#### 修改手机号

单独入口修改人员手机号，手机号不可与其他用户重复。

#### 重置密码

将用户密码重置为 `admin123`，该用户需重新登录并强制修改密码。

#### 禁用人员

将用户状态设为「封禁」，封禁后无法登录。不能禁用自己。

---

### 10.2 部门管理

组织架构为**两级结构**：

```
经营部（顶级）
├── 项目部 A
├── 项目部 B
└── 项目部 C
```

- **经营部**：顶级部门，不可挂在其他部门下
- **项目部**：必须隶属某个经营部，房源挂靠到项目部

支持部门树形展示、增删改查，部门下有人员或项目部时不可删除。

---

### 10.3 权限管理

通过**角色**控制用户的模块访问权限。

#### 角色管理

- 系统预置「系统管理员」角色（不可删除、不可修改模块权限）
- 支持创建自定义角色，配置可访问的模块
- 角色等级 `level` 用于数据权限层级判断

#### 模块权限

每个角色可配置能访问的菜单模块（如客户管理、房产管理等）。用户登录后，菜单仅显示其角色授权范围内的模块。

---

### 10.4 系统参数

管理系统级别的配置参数，如客户保护期天数（`CPP`，默认 30 天）等。

- 支持按参数名称、键值搜索
- **仅可修改参数值**，参数定义由系统初始化维护
- 不同角色可见不同范围的参数（系统管理员可见全部）

## 11 常见问题

### Q1：为什么我登录后看不到某些菜单？

A：菜单根据你的角色权限动态展示。如果缺少需要的功能模块，请联系管理员调整角色权限。

### Q2：为什么联系人电话显示为 `138****8000`？

A：系统对联系人手机号做了数据权限隔离。仅**你自己创建**的客户/联系人会显示完整号码；他人创建的联系人号码会脱敏展示。

### Q3：客户的「我的客户」变成了「公海」怎么办？

A：客户超过保护期（默认 30 天）未产生有效跟进，系统会自动将其转入公海。你可以在公海中重新认领该客户。

### Q4：修改密码后为什么被强制退出？

A：修改密码会影响账号安全，系统会吊销所有设备的登录会话，需重新登录。这是正常的安全机制。

### Q5：新员工初始密码是什么？

A：管理员新增人员时，初始密码固定为 `admin123`。首次登录会强制要求修改密码。

### Q6：房源价格想调回之前的版本怎么办？

A：在房源详情页可「回退」最新指导价/评估价到上一版本。批量调价支持整批次回退。

### Q7：签约后想撤销成交怎么办？

A：已签约的成交不可删除，但可以在修改页面调整信息。如果实际已退租，请使用「登记退租」功能。

### Q8：我可以同时在手机和电脑上登录吗？

A：可以。系统默认允许多端同时在线，互不影响。
